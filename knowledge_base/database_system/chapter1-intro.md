# 第一章：数据库系统概述

## 1.1 数据库的定义

数据库（Database）是按照数据结构来组织、存储和管理数据的仓库。它是长期存储在计算机内的、有组织的、可共享的数据集合。

## 1.2 数据库管理系统

数据库管理系统（DBMS）是用于管理数据库的软件系统，主要功能包括：
- 数据定义（DDL）：创建、修改、删除数据库对象
- 数据操纵（DML）：插入、更新、删除数据
- 数据查询（DQL）：检索数据
- 事务管理：保证数据的完整性和一致性

## 1.3 数据库系统的特点

1. **数据结构化**：数据之间存在联系，整体结构化
2. **数据共享性**：多个用户可以同时访问同一数据
3. **数据独立性**：数据与程序相互独立
4. **数据完整性**：保证数据的准确性和一致性
5. **数据安全性**：保护数据免受非法访问

## 1.4 数据库系统的组成

- **硬件**：存储数据库的物理设备
- **软件**：DBMS、操作系统、应用程序
- **数据**：数据库中的实际数据
- **人员**：数据库管理员、开发人员、用户

## 1.5 数据模型

常见的数据模型包括：
- 层次模型（Hierarchical Model）
- 网状模型（Network Model）
- 关系模型（Relational Model）
- 对象-关系模型（Object-Relational Model）
- NoSQL模型（如文档型、键值型、图模型）

## 1.6 关系数据库

关系数据库是基于关系模型的数据库，以表（Table）的形式组织数据：
- 表由行（Row/Tuple）和列（Column/Attribute）组成
- 每个表有一个主键（Primary Key）用于唯一标识行
- 表之间通过外键（Foreign Key）建立关系

## 1.7 数据库设计原则

1. **第一范式（1NF）**：列不可再分
2. **第二范式（2NF）**：非主键列完全依赖于主键
3. **第三范式（3NF）**：非主键列之间无传递依赖
4. **BCNF范式**：消除主属性对候选键的部分依赖

## 1.8 学习目标

通过本章学习，你将能够：
- 理解数据库和DBMS的基本概念
- 了解数据库系统的组成和特点
- 掌握关系模型的基本原理
- 理解数据库设计的规范化原则
